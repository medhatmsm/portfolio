# --- Precision-Recall Curve ---
# 1. Get prediction probabilities (if y_hat contains them directly)
# Assuming y_hat contains the probabilities for the positive class (from sigmoid activation)
# If y_hat is multi-dimensional (e.g., (n_samples, 1)), flatten it to (n_samples,)
y_scores = y_hat.flatten() if isinstance(y_hat, np.ndarray) and y_hat.ndim > 1 else np.array(y_hat).flatten()

# Ensure y_test is a flat numpy array of integers (0 or 1)
y_true = np.array(y_test).flatten()

# 2. Compute Precision-Recall pairs for different probability thresholds
precision, recall, thresholds = precision_recall_curve(y_true, y_scores)

# Calculate the Area Under the Precision-Recall Curve (AP)
avg_precision = average_precision_score(y_true, y_scores)

# Calculate F1-score for each threshold
# Note: F1 is undefined where precision + recall is 0, so handle division by zero
f1_scores = (2 * precision * recall) / (precision + recall + 1e-10) # Added epsilon for stability

# Find the threshold that maximizes the F1-score
optimal_f1_idx = np.argmax(f1_scores)
optimal_threshold = thresholds[optimal_f1_idx]
max_f1_score = f1_scores[optimal_f1_idx]

print(f"\nOptimal Threshold (maximizing F1-score): {optimal_threshold:.4f}")
print(f"Max F1-score at optimal threshold: {max_f1_score:.4f}")


# 3. Plot the Precision-Recall Curve
plt.figure(figsize=(9, 7))
plt.plot(recall, precision, color='darkorange', lw=2, label=f'Precision-Recall curve (AP = {avg_precision:.2f})')
plt.xlabel('Recall')
plt.ylabel('Precision')

After applying class weighting, we observe an important shift in model's behavior:
1. **Initial Bias ** Previously, the model favored predicting 'No Churn' (the majority class), leading to high overall accuracy but missing many actual churners.

3. **New Balance (Precision-Recall Curve):** This curve shows the trade-off between finding all 'Actual Churn' customers (Recall) and ensuring your 'Predicted Churn' customers are truly churners (Precision). The 'Optimal point' on this curve indicates the best balance (highest F1-score) for business needs.
4. **Improved Churn Detection (Final Confusion Matrix):** Comparing the 'Predicted Churn' column in this final Confusion Matrix (at the optimal threshold) with the very first one, we see a higher number of 'Actual Churn' customers now correctly predicted as 'Churn'. This demonstrates that the model is now better at identifying the key group of customers you want to target for retention efforts, even if it means a slight increase in 'false alarms' (predicting churn when there isn't any).
